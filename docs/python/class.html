    # 首先明确什么是类？

    类是一种抽象的数据结构,它包含了你所需要实体化对象的共有特征.

    # 类可以用来干什么？

    类最大的用处在于可以实体化对象,减少重复.以及它可以减少代码的复用.实例化的对象都拥有类中所有定义的方法和属性 比如

        class Dog():
        def __init__(self, name , age ):
            self.name = name;
            self.age = age;

        def sit(self):
            printf(self.name.title() + " is now sitting.");

    a = Dog("a","15");
    b = Dog("b","16");

    1. 类的属性访问

      • 类的属性访问有几种不同的办法.首先,直接访问.例如刚才的a.name,可以直接访问和直接更改.其次,可以在定义类的时候,定义对应的方法,使用对应的方法进行访问和更改.例如
          def getName(self):
        return self.name;

      2

      直接使用a.getName()就可以获得Dog类的中的name属性(使用方法更改属性和这里同理)

    2. 类的方法使用

      • 可以直接使用实例化之后的对象访问.a.sit();

    3. 继承

      • 首先明确什么是继承.继承是一个类需要另外一个类的东西,通过继承来获取.比如electricCar类可以继承car类(car类是所有车类的共同属性和方法).
      • 首先需要注意的是利用super的时候,调用继承类的__init__,第一个参数就不是self.以及继承类在定义下一个类时,作为参数传入.
          class Car():
        def __init__(self,make,model,year):
            self.make = make;
            self.model = model;
            self.year = year;


    class ElectricCar(Car): // 这里注意作为参数传入
        def __init__(self,make,model,year):
            super().__init__(make,model,year); // 这里注意不要传入self

    4. 重写

      • 首先明确什么是重写.重写是在子类中重新写与父类同名的方法.当子类调用该方法时,就会调用使用的方法,不会使用父类的方法.

    :::tips 可以使用类套类的方法,将对应的方法独立出来.比如一个电动自行车,它既包含电动的属性也包含自行车的属性.所以它在初始化时可以写为 it.elec = Electric(); it.bic = Bicycle(); 这样写的好处在于不用将全部的属性放入it中,从而导致属性过多引发的混乱.这样写可以保证你在寻找对应的方法的时候快捷并且准确.如果你想访问它的自行车属性,直接对它的bic访问就好,使用it.bic.attributeName,就可以访问了,简化因为属性过多导致混乱的情况. :::

    1. 导入

      • 为什么需要导入？是因为我们需要把类封装到一个文件内,在另外一个文件中需要引用相关的类.
          from fileName import ClassName
      1

      fileName表示是需要引用的类所在文件,className是需要引用的类的名字(注意如果需要引用的类是多个,则用逗号隔开即可)

      如果是需要引用整个模块,则:

          import fileName;
      1

      调用则是:

          fileName.ClassName()
      1

      当不同的类相互引用的时候,你只需要引用最后所需要的那个类就好.比如B引用了A,C引用了B,现在你要用C,那你就直接引用C,不需要引用之前的A和B.
